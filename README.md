# 마이크 입력 기반 한국어 표준 발음 분석기

## 1. 한 줄 결론

마이크 입력 또는 직접 입력으로 받은 한국어 표기형 단어에 규칙 기반 G2P 알고리즘을 적용해 표준 발음과 음운 변동 과정을 보여 주는 학습용 웹앱입니다.

## 2. 핵심 원리

이 프로젝트는 음성 자체를 바꾸는 프로그램이 아닙니다. 마이크로 들어온 한국어 음성은 브라우저의 Web Speech API를 통해 먼저 텍스트로 인식됩니다. 이후 인식된 표기형 텍스트를 한글 유니코드 계산 방식으로 초성, 중성, 종성으로 분해하고, 국어 음운 변동 규칙을 조건문처럼 적용합니다.

한글 음절은 다음 공식으로 분해하고 다시 조합합니다.

```text
한글 시작 코드 = 0xAC00
한 음절 코드 = 0xAC00 + 588 * 초성 index + 28 * 중성 index + 종성 index
```

예를 들어 `국물`은 첫 음절 `국`의 받침 `ㄱ` 뒤에 다음 음절 초성 `ㅁ`이 이어지므로 비음화가 적용됩니다. 그래서 `ㄱ`이 `ㅇ`으로 바뀌어 `궁물`로 분석됩니다.

```text
국물 → 궁물
받침 ㄱ 뒤에 비음 ㅁ이 이어져 ㄱ이 ㅇ으로 바뀜
```

## 3. 구현 구조

전체 흐름은 다음과 같습니다.

```text
마이크 입력 또는 직접 입력
→ normalizeInput()
→ decomposeHangul()
→ applyG2PRules()
→ composeHangul()
→ 표준 발음과 분석 과정 출력
```

마이크 입력의 경우에는 다음 순서로 동작합니다.

```text
마이크로 입력하기 버튼
→ 브라우저 마이크 권한 요청
→ recognition.lang = "ko-KR"로 한국어 음성 인식
→ STT 결과를 텍스트로 표시
→ G2P 규칙 적용
→ 표준 발음, 적용 규칙, 분석 과정 표시
```

## 4. 파일별 역할

```text
index.html
styles.css
src/
  app.js
  g2p.js
  hangul.js
  stt.js
```

- `index.html`: 제목, 입력 영역, 결과 영역, 한계 안내 등 화면의 의미 구조를 담당합니다.
- `styles.css`: ThoughtStream 디자인 기준에 맞춘 색상, 여백, 반응형 레이아웃을 담당합니다.
- `src/hangul.js`: 한글 음절을 초성, 중성, 종성으로 분해하고 다시 조합합니다.
- `src/g2p.js`: 비음화, 유음화, 구개음화, 된소리되기, 단순 연음 규칙을 적용하고 테스트 케이스를 제공합니다.
- `src/stt.js`: Web Speech API를 이용한 한국어 음성 인식 기능을 담당합니다.
- `src/app.js`: 버튼 클릭, 입력값 검증, 분석 결과 렌더링을 연결합니다.

## 5. 코드 핵심 부분

### `decomposeHangul(syllable)`

한글 한 음절을 초성, 중성, 종성으로 나눕니다. 유니코드에서 `가`를 기준점으로 삼고, 몇 번째 초성·중성·종성인지 계산합니다.

### `composeHangul(initial, medial, final)`

초성, 중성, 종성 값을 다시 하나의 완성형 한글 음절로 조합합니다.

### `normalizeInput(text)`

입력값의 앞뒤 공백과 분석에 필요 없는 일부 특수문자를 정리합니다. 현재는 완성형 한글 음절 중심의 입력을 지원합니다.

### `analyzePronunciation(text)`

입력 단어를 받아 정규화, 한글 분해, 규칙 적용, 결과 생성을 한 번에 수행하는 대표 함수입니다.

### `applyG2PRules(syllables)`

분해된 음절 배열에 구개음화, 비음화, 유음화, 된소리되기, 단순 연음 규칙을 순서대로 적용합니다. 규칙이 적용될 때마다 다음 정보를 로그로 남깁니다.

```js
{
  rule: "비음화",
  before: "국물",
  after: "궁물",
  explanation: "받침 ㄱ 계열 뒤에 비음 ㅁ이 이어져 ㄱ 계열 받침이 ㅇ으로 바뀌었습니다."
}
```

## 6. 실행 방법

이 프로젝트는 JavaScript 모듈을 사용하므로 `index.html`을 직접 여는 것보다 로컬 서버로 실행하는 편이 안전합니다.

PowerShell에서 프로젝트 폴더를 연 뒤 실행합니다.

```powershell
& "C:\Users\sol2508\.cache\codex-runtimes\codex-primary-runtime\dependencies\python\python.exe" -m http.server 5173
```

브라우저에서 다음 주소를 엽니다.

```text
http://localhost:5173
```

사용 순서:

1. 직접 입력창에 `국물` 같은 단어를 입력합니다.
2. `직접 입력 분석하기` 버튼을 누릅니다.
3. 인식된 텍스트, 표준 발음, 적용된 음운 변동, 분석 과정을 확인합니다.
4. Chrome 또는 Edge에서는 `마이크로 입력하기` 버튼으로 음성 인식도 시도할 수 있습니다.

## 7. 테스트 방법

브라우저 개발자 콘솔에서 다음 명령을 실행합니다.

```js
console.table(runG2PTests());
```

모든 행의 `passed` 값이 `true`이면 핵심 테스트가 통과한 것입니다.

직접 입력으로 확인할 테스트 단어는 다음과 같습니다.

| 입력 | 기대 발음 | 규칙 |
| --- | --- | --- |
| 국물 | 궁물 | 비음화 |
| 밥물 | 밤물 | 비음화 |
| 닫는 | 단는 | 비음화 |
| 신라 | 실라 | 유음화 |
| 난로 | 날로 | 유음화 |
| 굳이 | 구지 | 구개음화 |
| 같이 | 가치 | 구개음화 |
| 국밥 | 국빱 | 된소리되기 |
| 먹다 | 먹따 | 된소리되기 |
| 먹어 | 머거 | 단순 연음 |
| 밥이 | 바비 | 단순 연음 |
| 꽃이 | 꼬치 | 단순 연음 |

개발 환경에서 알고리즘만 빠르게 확인하려면 다음 명령을 사용할 수 있습니다.

```powershell
& "C:\Users\sol2508\.cache\codex-runtimes\codex-primary-runtime\dependencies\node\bin\node.exe" --input-type=module -e "import('./src/g2p.js').then(({ runG2PTests }) => { const results = runG2PTests(); console.table(results); if (results.some((item) => !item.passed)) process.exit(1); })"
```

## 8. 오류 발생 시 해결 방법

- 마이크 권한이 거부된 경우: 브라우저 주소창의 권한 설정에서 마이크를 허용하거나 직접 입력 기능을 사용합니다.
- Web Speech API를 지원하지 않는 경우: Chrome 또는 Edge를 사용하거나 직접 입력 기능을 사용합니다.
- 빈 입력을 넣은 경우: 분석할 한국어 단어를 입력한 뒤 다시 실행합니다.
- 한글이 아닌 입력을 넣은 경우: 현재는 완성형 한글 입력만 지원합니다.
- `index.html`을 직접 열었을 때 모듈 오류가 나는 경우: 로컬 서버 주소인 `http://localhost:5173`으로 접속합니다.

## 9. 모바일 테스트 방법

같은 Wi-Fi에 연결된 스마트폰에서 테스트하려면 PC의 로컬 IP 주소를 확인한 뒤 다음 형식으로 접속합니다.

```text

```

주의할 점:

- 모바일 브라우저에서 마이크 권한을 허용해야 합니다.
- 일부 모바일 브라우저는 Web Speech API를 지원하지 않을 수 있습니다.
- 마이크 입력이 안 되더라도 직접 입력 분석은 사용할 수 있습니다.
- 마이크 권한은 브라우저와 운영체제 설정의 영향을 받습니다.

## 10. 한계점

이 프로그램은 국어 음운 변동 학습을 위한 규칙 기반 G2P 프로토타입입니다. 실제 상용 한국어 발음 변환기처럼 모든 단어와 문장을 처리하지는 않습니다. 특히 구개음화, ㄴ첨가, 예외 발음 등은 형태소 분석기와 표준 발음 사전이 필요할 수 있습니다.

단순 연음도 같은 이유로 제한적으로만 구현했습니다. 이 프로그램의 단순 연음은 `먹어 → 머거`, `밥이 → 바비`, `꽃이 → 꼬치`처럼 앞 음절 받침을 뒤 음절의 빈 초성 `ㅇ` 자리로 옮기는 기본 사례를 설명하기 위한 규칙입니다. 하지만 `읽어`, `값이`, `닭이`처럼 겹받침이 포함된 연음은 구현하지 않았습니다. 실제 표준 발음에서는 겹받침이 있는 경우 어느 자음이 남고 어느 자음이 뒤로 이어지는지 별도 규칙이 필요합니다. 또한 `이`가 조사인지, 어미인지, 단어 내부의 일부인지 판단하려면 형태소 경계 정보가 필요합니다. `같이 → 가치`처럼 겉보기에는 연음처럼 보여도 실제로는 구개음화가 우선되는 경우도 있으므로, 규칙 적용 순서와 예외 발음 사전도 중요합니다.

실제 시스템과 이 프로젝트의 차이는 다음과 같습니다.

- 실제 G2P 시스템: 형태소 분석, 발음 사전, 예외 처리, 문맥 분석을 함께 사용합니다.
- 이 프로젝트: 대표 예시를 설명하기 위한 규칙 기반 조건문으로 동작합니다.
- 실제 음성 처리 앱: 음성 신호 자체를 분석하거나 합성할 수 있습니다.
- 이 프로젝트: 음성은 STT로 텍스트화할 뿐이며, 실제 음성을 변형하지 않습니다.

또한 ThoughtStream 디자인을 위해 Google Fonts를 연결했지만, 인터넷 연결이 없으면 브라우저 기본 대체 글꼴로 표시됩니다.

## 11. 다음 개선 방향

- 겹받침이 포함된 연음 규칙 확장
- 형태소 경계 정보를 활용한 연음 처리
- ㄴ첨가, 겹받침 단순화, ㅎ 탈락 등 규칙 확장
- 형태소 분석기 또는 표준 발음 사전 연동
- 문장 단위 분석과 단어별 분석 로그 분리
- 테스트 결과를 화면에서 바로 실행하는 기능 추가
- 모바일 마이크 인식 호환성 안내 강화

## 12. 탐구 의의

국어 음운 변동 규칙을 If-Then 조건문으로 구조화하여, 국어 문법의 규칙성을 컴퓨터 알고리즘으로 표현했다. 또한 마이크 입력, STT, G2P 변환을 결합하여 국어 교과 지식이 음성 인터페이스 기술과 연결될 수 있음을 확인하는 것을 목표로 한다.

## 13. 활용 또는 정리 가능한 요약문

본 프로젝트는 한국어 표기형 단어를 초성, 중성, 종성으로 분해한 뒤 국어 음운 변동 규칙을 순서대로 적용하여 표준 발음을 계산하는 웹 기반 학습 도구이다. 이를 통해 국어 문법의 규칙성이 컴퓨터 알고리즘의 조건문과 자료 구조로 표현될 수 있음을 확인했다. 마이크 입력과 STT를 함께 사용함으로써 국어 지식이 음성 인터페이스 기술과 연결될 수 있음을 탐구했다.
